Privacy Policy

YNAB Spending Dashboard · Last updated 2026-04-28

Short version: The dashboard sends your YNAB data straight from your browser to YNAB and back. The dashboard server never sees your budget, your transactions, or your YNAB credentials. The only thing routed through the server is the OAuth token exchange itself, and even that is forwarded without inspection or logging.

What this site is

The YNAB Spending Dashboard (ynab-dashboard.org) is an open-source, browser-based visualization tool for personal finance data stored in You Need A Budget (YNAB). The full source code is available at github.com/TnTBass/ynab-spending-dashboard.

Data we never collect

  • Your YNAB email, password, or login credentials. The dashboard never sees these. You authorize via YNAB's own OAuth screen.
  • Your YNAB budget data, transactions, categories, account balances, or any other financial information. These are fetched from api.ynab.com directly by your browser and stay in your browser.
  • Cookies, analytics, advertising identifiers, fingerprints, or any third-party tracking.
  • Your IP address, beyond the standard request logs that any web host (Cloudflare in this case) keeps for short periods to operate and protect the service.

What is stored, and where

  • OAuth access and refresh tokens are stored in your browser's localStorage after you sign in with YNAB. These let the dashboard call the YNAB API on your behalf without re-prompting. They live only in your browser and are never sent to the dashboard server, except via the token-refresh proxy described below.
  • Your selected budget ID and a small client-side cache of YNAB API responses are also stored in your browser, in localStorage and sessionStorage. This is a performance optimization, not a tracking mechanism.
  • You can clear all of this at any time by clicking Sign out in the dashboard, by clearing your browser's site data for ynab-dashboard.org, or by revoking the application from your YNAB OAuth settings.

What the server does

The dashboard runs on Cloudflare Workers. The server has exactly three responsibilities:

  1. Serving the static HTML, CSS, and JavaScript that make up the dashboard. These are public files identical to what you see in the GitHub repo.
  2. Returning the public OAuth client ID to the dashboard JavaScript on first load (GET /api/config). This value is not sensitive.
  3. Proxying two OAuth requests to YNAB:
    • POST /api/oauth/exchange — trades an authorization code for an access token plus a refresh token, immediately after you sign in
    • POST /api/oauth/refresh — trades an expiring access token for a fresh one
    Both are forwarded to YNAB's /oauth/token endpoint with the OAuth client secret attached. The proxy exists only because YNAB requires the client secret on the token exchange, and a browser cannot safely hold one. The server forwards the token responses verbatim back to your browser. The server does not log, store, inspect, or persist token bodies or YNAB API responses.

Network requests your browser makes while using the dashboard

  • To ynab-dashboard.org — loads the dashboard HTML/JS, fetches the OAuth client ID, and proxies the two OAuth token requests.
  • To app.ynab.com — the OAuth authorize redirect when you click "Sign in with YNAB".
  • To api.ynab.com — every call to read your budget, transactions, and categories. Sent directly from your browser using your access token.
  • To cdn.jsdelivr.net and cdnjs.cloudflare.com — loads three open-source JavaScript libraries from public CDNs: Chart.js for the charts, plus html2canvas and jsPDF for the PDF export feature. These CDNs may receive standard request metadata (your IP address, user-agent) when serving these files.

Hosting and infrastructure

The dashboard is hosted on Cloudflare Workers. Cloudflare keeps standard request logs (timestamp, IP address, user-agent, response status) as part of operating the service. These logs do not contain OAuth tokens, your YNAB data, or any other request bodies. Cloudflare's own privacy practices are described at cloudflare.com/privacypolicy.

YNAB's role

This dashboard uses the YNAB API. When you authorize the dashboard, you are granting it read-only access to your budget. This means the dashboard cannot create, modify, or delete anything in your YNAB account. YNAB enforces this restriction on their end. Your relationship with YNAB and the data they hold about you is governed by YNAB's privacy policy.

Data we do not share

Because the dashboard server does not see, store, or log your budget data or tokens, there is nothing for the operator of this dashboard to share with anyone. No analytics, no advertising partners, no third-party data sales, no aggregated reporting on user behavior.

Children

This service is not directed to children under 13 and we do not knowingly collect any personal information from them. (We do not knowingly collect personal information from anyone — see above — this clause is included for the sake of completeness.)
